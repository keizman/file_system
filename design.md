APK文件管理系统设计说明文档
===============

1\. 系统概述
--------

本系统采用Client-Server架构，用于管理和快速查找分布在多个文件服务器上的APK文件。Server端负责扫描、索引和提供文件服务，Client端提供友好的图形界面进行文件查找、下载和管理。 code 和 程序控件内容为英文

3\. Server端设计
-------------

### 3.1 环境配置 (.env)

    FILE_SERVER_1_USER=smbuser
    FILE_SERVER_1_PASS=123456
    
    FILE_SERVER_2=\\192.168.1.37\yw\apks

    # 更新间隔
    UPDATE_INTERVAL=5m

    SERVER_PORT=9301
    
    # 日志级别

### 3.2 核心功能

#### 3.2.1 文件扫描服务

*   **增量扫描算法**：通过检查二级目录数量变化来判断是否需要扫描
*   **并发扫描**：支持多线程扫描提高效率
*   **文件过滤**：只扫描.apk文件
*   **元数据提取**：文件大小、创建时间、修改时间、MD5（可选）
*  **启动时/定时/手动触发**：扫描任务启动。
*  **连接服务器**：遍历 `FILE_SERVERS` 配置中的每个服务器。
*  **遍历二级目录**：对于每个服务器，列出其根路径下的所有二级目录（如 `App_UvFree`）。
*  **检查更新**：
    *   获取当前二级目录下的子目录数量。
    *   与 Redis 中 `apk_finder:<server_name>:meta` 存储的数量进行比较。
    *   **如果数量一致**：跳过此二级目录，极大提升效率。
    *   **如果数量不一致或首次扫描**：
        *   深度递归扫描此二级目录下的所有 `.apk` 文件。
        *   收集每个 APK 的元数据（相对路径、大小、修改时间）。
        *   将收集到的元数据列表更新到 Redis 的 `apk_finder:<server_name>` HASH 中。
        *   更新 `apk_finder:<server_name>:meta` 中该二级目录的子目录数量。
            
5.  **清理旧数据**：扫描完成后，可以检查 Redis 中存在但文件服务器上已不存在的二级目录，并将其从 Redis 中移除。

#### 3.2.2 Redis存储结构

    {
      "key": "App_UvFree",  // 二级目录名作为key
      "value": [
        {
          "relative_path": "\\App_UvFree\\20250707_18\\signUnitvS\\release\\uvFree_P723_V1_81304fc9_20250707_18_beta_jiagu.apk",
          "file_name": "uvFree_P723_V1_81304fc9_20250707_18_beta_jiagu.apk",
          "file_size": 104857600,  // 字节
          "created_time": "2025-07-07T18:00:00",
          "server_prefix": "\\\\192.168.1.39\\build\\Apps",
          "build_type": "release",  // release/debug
		  "download_time": 1, 触发下载的次数, 触发下载时更新
          "md5": "xxxxxxxxxxxx", 文件 md5 用于给 client 校验完整性, 第一次下载到 server 本地后更新
        }
      ]
    }
    

### 3.3 API接口设计

#### 3.3.1 搜索接口

    POST /api/search
    {
      "keyword": "1454515",
      "server": "192.168.1.39",  // 可选
      "build_type": "release",   // release/debug/combine
      "limit": 10,
      "offset": 0
    }
    
    Response:
    {
      "code": 200,
      "data": {
        "total": 50,
        "items": [...]
      }
    }
    

#### 3.3.2 刷新接口

    POST /api/refresh

    

#### 3.3.3 下载接口

    GET /api/download?path=xxx&server=xxx
    

#### 3.3.4 文件信息接口

    GET /api/file/info?path=xxx
    

#### 3.3.5 系统状态接口

    GET /api/status
    Response:
    {
      "last_scan_time": "2025-01-17T10:00:00",
      "total_files": 1000,
      "scanning": false
	  ....
    }
    

### 3.4 缓存策略

*   **下载缓存**：下载的文件临时存储在./tmp目录
*   **预热机制**：可配置热门文件预下载

### 3.5 错误处理

*   **网络异常重试**：文件服务器连接失败时的重试机制
*   **降级策略**：当某个文件服务器不可用时的处理
*   **日志记录**：详细的错误日志和操作日志

### 4.1.3 界面美化 (QSS)

*   **配色**:
    
    *   **主背景**: 白色 (`#FFFFFF`) 或非常浅的灰色 (`#F5F5F7`)。
        
    *   **主色调 (蓝)**: 用于 Tab 选中状态、按钮、标题等，选择一种现代感的蓝色 (如 `#3B82F6`)。
        
    *   **点缀色 (紫)**: 用于悬停效果、少量图标或重点提示，选择柔和的紫色 (如 `#8B5CF6`)。
        
*   **控件**:
    
    *   所有控件（按钮、输入框）使用圆角设计。
        
    *   添加细微的阴影效果，增加层次感。
        
    *   自定义 `QTableWidget` 的滚动条样式。
        
    *   使用清晰的字体（如 "Inter", "Segoe UI", "Microsoft YaHei"）。
### 4.2 功能设计

#### 4.2.1 右键菜单

*   复制SMB路径：`\\192.168.1.37\yw\apks\filepath\file.apk`
*   复制HTTP路径：`http://192.168.1.37/yw/apks/filepath/file.apk`
*   下载到本地, 根据 server 端下载完成时记录的 md5 校验 (记住作为独立函数,方便随时关闭校验)
*   查看文件详情
*   auto install(need adb), 下载到本地后直接安装, 如果 adb devices 未显示 设备, 功能置灰, 如果有 一个 device 即可使用, 如果 超过一个 device, 则所有device 展示在二级菜单, 点击后 执行 adb -s serial install -r -d -t "local file path"

#### 4.2.2 配置管理

*   Server地址配置
*   默认下载路径
*   搜索历史记录
*   界面主题切换

### 4.3 客户端缓存

*   搜索结果缓存
*   下载历史记录 on download
*   常用文件收藏

### 4.4 搜索APK文件
* 支持单次匹配, 直接输入内容, 支持多次匹配, 通过 | 字符隔开, 即需要匹配前后两者. (注意 若在 release / debug 下 匹配时默认添加 release 匹配) comb 则不添加额外匹配项
* 搜索框右侧有 search button 点击时触发对应搜索条件的查询

### 4.5 文件列表

* 列表默认展示 10 行内容 (配置中心可调整) 
* 切换 debug / release / comb tab 时更新 列表为对应搜索条件的 文件, 按 create time 倒序排序(通过接口传递所需内容到 sever 即可, 其会返回符合条件的内容)
* 无符合条件的内容时展示对应提示

6\. 技术栈建议
---------

### Server端

*   **语言**：Python 3.8+
*   **Web框架**：FastAPI（高性能、自动API文档）
*   **异步框架**：asyncio + aiofiles
*   **Redis客户端**：redis-py
*   **SMB客户端**：smbprotocol 或 pysmb
*   **定时任务**：APScheduler
*   **日志**：loguru

### Client端 for win

*   **GUI框架**：PyQt5 或 Kivy（更现代的选择）
*   **HTTP客户端**：httpx（支持异步）
*   **图标库**：QtAwesome
*   **打包工具**：PyInstaller
